    MÉTODO PRINCIPAL — func run(url: String, outputType: String, selector: String? = nil, timeout: Int = 15) async throws -> ExtractionResult:

    1. VALIDACIÓN PREVIA: Verificar que pythonPath no está vacío y que FileManager.default.isExecutableFile(atPath: pythonPath) devuelve true. Si no, throw ExtractionError.pythonNotFound(path: pythonPath). Verificar que scriptPath no está vacío y que FileManager.default.fileExists(atPath: scriptPath) es true. Si no, throw ExtractionError.pythonNotFound(path: scriptPath).

    2. CONFIGURACIÓN DEL PROCESO: Construir el Process() con:
       - executableURL = URL(fileURLWithPath: pythonPath)
       - arguments: [scriptPath, url, "--type", outputType, "--json"] más ["--selector", sel] si selector no es nil, más ["--timeout", "\(timeout)"]
       - Pipes: let stdoutPipe = Pipe() y let stderrPipe = Pipe() asignados a standardOutput y standardError

    3. ENTORNO EXPLÍCITO (crítico — no heredar el PATH de launchd):
       - Calcular scriptDir = URL(fileURLWithPath: scriptPath).deletingLastPathComponent().path
       - Calcular venvBin = scriptDir + "/.venv/bin"
       - Partir de var env = ProcessInfo.processInfo.environment
       - Inyectar: env["VIRTUAL_ENV"] = scriptDir + "/.venv"
       - Inyectar: env["PATH"] = venvBin + ":" + (env["PATH"] ?? "/usr/bin:/bin:/usr/local/bin")
       - Asignar: process.environment = env
       - Asignar: process.currentDirectoryURL = URL(fileURLWithPath: scriptDir)

    4. LANZAR EL PROCESO: Usar do { try process.run() } catch { throw ExtractionError.processLaunchFailed(underlying: error) }

    5. LEER PIPES EN PARALELO (patrón crítico — no leer secuencialmente):
       - async let outData: Data = Task.detached { stdoutPipe.fileHandleForReading.readDataToEndOfFile() }.value
       - async let errData: Data = Task.detached { stderrPipe.fileHandleForReading.readDataToEndOfFile() }.value
       - let (stdout, _) = try await (outData, errData)
       - Llamar process.waitUntilExit() DESPUÉS de await (nunca antes, y nunca desde @MainActor)

    6. DECODIFICAR: Si stdout está vacío, throw ExtractionError.emptyOutput. Intentar JSONDecoder().decode(ExtractionResult.self, from: stdout); si falla, throw ExtractionError.jsonDecodeFailed(underlying: error). Si result.isSuccess es false, throw ExtractionError.extractionFailed(message: result.errorMessage ?? "Error desconocido"). Devolver result.

    MÉTODO DE TEST — func testRun() async throws -> ExtractionResult: llama a run(url: "https://example.com", outputType: "text", selector: nil, timeout: 10). Permite verificar el bridge desde ContentView sin UI.

    ADVERTENCIA CRÍTICA: no usar waitUntilExit() antes de haber await-ado ambas pipes. No leer stdout hasta EOF y después stderr (o al revés). El orden correcto es: run() → async let outData + async let errData → await ambas → waitUntilExit().

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process.environment → subprocess Python | Variables de entorno controladas pasadas al proceso hijo |
| stdout del proceso hijo → JSONDecoder | Datos no confiables del proceso hijo se deserializan a tipos Swift |
| Rutas configuradas por el usuario → FileManager | Paths arbitrarios del usuario verificados antes de ejecutar |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03 | Elevation of Privilege | process.environment PATH injection | mitigate | PATH se construye explícitamente: venvBin + ProcessInfo.environment["PATH"]. No se acepta PATH de launchd sin control; no hay interpolación de strings sin validar |
| T-03-04 | Denial of Service | Pipe buffer deadlock | mitigate | async let en paralelo para stdout y stderr; waitUntilExit() solo después de await de ambas — patrón verificado en RESEARCH.md sección 3 |
| T-03-05 | Tampering | JSON stdout decodificado sin validar status | mitigate | Verificar result.isSuccess después de decode; si status == "error" propagar extractionFailed con el mensaje — nunca silenciar |
| T-03-06 | Repudiation | Errores de proceso silenciados | mitigate | Todos los casos de ExtractionError son throws; el caller siempre recibe el error explícito |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | Este plan no instala paquetes — solo crea ficheros Swift |
</threat_model>
