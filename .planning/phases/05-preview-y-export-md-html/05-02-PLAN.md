<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| contenido extraído → WKWebView | `htmlForPreview` (derivado de contenido no confiable) se carga en WKWebView vía loadHTMLString |
| SwiftUI ↔ AppKit (Coordinator) | El Binding contentReady cruza entre el delegate AppKit y el estado SwiftUI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | WebPreviewView — loadHTMLString con contenido extraído | mitigate | `loadHTMLString(html, baseURL: nil)` asigna origen about:blank; sin acceso a cookies/almacenamiento del dominio real ni fetch externo (sin WKWebViewConfiguration de content rules). El escape de backticks/HTML ya se aplica en generateHTML (Plan 01, T-05-01). marked.js solo convierte Markdown→HTML. |
| T-05-04 | Denial of Service | updateNSView — recarga en cada re-render | mitigate | Guard `htmlContent != lastLoadedHTML` (PITFALL 1) evita el ciclo de recarga/parpadeo que dispararía didFinish repetidamente y resetearía contentReady en cada hover. |
| T-05-02 | Tampering | NSSavePanel nameFieldStringValue | accept | Heredado del Plan 01: el nombre sugerido es solo un valor por defecto; el usuario elige la ruta final. Herramienta personal de un usuario. |
</threat_model>
