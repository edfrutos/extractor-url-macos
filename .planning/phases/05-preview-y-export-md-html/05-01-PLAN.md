<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| contenido extraído → template HTML/JS | El `resultContent` proveniente del sitio web cruza a un template JS (`marked.parse(\`...\`)`); entrada no confiable interpolada en código |
| ViewModel → sistema de archivos | `exportMarkdown`/`exportHTML` escriben en una ruta elegida por el usuario vía NSSavePanel |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | generateHTML — interpolación de resultContent en template JS | mitigate | Escapar `\`, `` ` `` y `$` EN ESE ORDEN antes de interpolar en `marked.parse(\`...\`)` (PITFALL 3). marked.js solo convierte Markdown→HTML, no ejecuta scripts del contenido original. HTML output (`outputType=="html"`) ya está saneado por el extractor Python. |
| T-05-02 | Tampering | exportMarkdown/exportHTML — nameFieldStringValue NSSavePanel | accept | El nombre sugerido (`suggestedFilename`) es solo un valor por defecto del campo; el usuario elige libremente la ruta final y NSSavePanel gestiona el path. Herramienta personal de un solo usuario — sin restricciones adicionales. |
| T-05-03 | Information Disclosure | template HTML exportado | mitigate | `generateHTML` usa CSS y marked.js inline, `loadHTMLString(baseURL: nil)` → origen about:blank, sin fetch de recursos externos. Test `testGenerateHTMLNoExternalDeps` verifica ausencia de `src="http`, `<link rel`, `@import url(http`. |
| T-05-SC | Tampering | npm install marked@18.0.5 (solo en build, no en producción) | mitigate | marked@18.0.5 verificado en RESEARCH.md Package Legitimacy Audit (npm registry, repo oficial markedjs/marked, archivo UMD 42 KB). El JS se bundlea como constante Swift — no es dependencia de runtime. Disposición: aprobado (no [ASSUMED]/[SUS], no requiere checkpoint humano). |
